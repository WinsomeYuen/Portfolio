Plan Sensitivity

The experiments team ran a short A/B test to redirect ineligible users to our /premium page to view all our plans. One functionality they wanted us to add was a banner to display the plan the user was on - if they were on one, and to disable ability to purchase. This required some changes to our components and extra logic to be added in Symfony serving our premium page. I've put a selection of some of the projects below which I can talk about in more detail than other projects to give you a taste of the type of work I do as a web engineer.

  • Support ineligibility messaging - extended functionality of the component used to display plans to support the error message banner. Made the changes required in Symfony to receive the response from checkout to render the error message.
  • Added the "Your Current Plan" banner to our flavour pages - extended the components in our library to support showing the “Your Current Plan” banner and remove functionality to purchase the plan. Changes made to Symfony to query users' current plan to pass this information to the components.
  • Documentation - handoff work to Technical Execution Managers on how to configure the new fields in the components.
